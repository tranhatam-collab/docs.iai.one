What Is IAI

IAI is a connected system of workflow execution, communication delivery, operator control, and core infrastructure. This docs site is the single place to understand how those surfaces fit together.

AI Workflow Engine

flow.iai.one provides the visual builder, node graph, execution runtime, and automation layer.

Communication Layer

mail.iai.one handles transactional email, notifications, and system alerts coming out of flows and services.

Core Infrastructure

home.iai.one frames the overall platform vision, while app.iai.one becomes the operator-facing control surface.

System Overview

The runtime path is designed to be simple to understand for new users and explicit enough for developers to deploy quickly.

User Creates input, configures a workflow, triggers execution.
→
Flow Builder Canvas UI for nodes, edges, drafts, preview, and validation.
→
API Gateway, auth, session handling, and platform contracts.
Runtime Execution engine, queue dispatch, retry logic, and logs.
→
Node System Definitions, execution mapping, and future AI-capable nodes.
→
Output Stored results, downstream actions, webhooks, and notifications.
Mail Layer Side channel for sending output summaries, alerts, and transactional messages.
App Surface Operator workspace for human review, dashboards, and workflow operations.
