Planned Automation Surface

flows.iai.one

flows.iai.one is intended to be the n8n-backed automation surface. As of March 28, 2026, the hostname is not resolving publicly, so this page documents readiness requirements and naming boundaries rather than a live system.

Intended Role

Automation Ops

Hold n8n-style automation flows, connectors, and external orchestration that do not belong inside the Flow builder UI.

Webhook Fabric

Receive or trigger automation events across third-party systems where low-code routing matters.

Operational Boundary

Stay distinct from flow.iai.one, which should remain the first-class workflow product surface.

Naming Boundary: Flow vs Flows

Keep the distinction explicit

flow.iai.one is the workflow builder and runtime product. flows.iai.one should only exist if it clearly represents an automation or orchestration environment such as n8n. Otherwise the names will confuse users and developers.

Go-Live Checklist

  • DNS live and public auth flow defined.
  • Ownership boundary documented against flow.iai.one and api.flow.iai.one.
  • Webhook ingress and egress rules documented.
  • Secrets, credentials, and admin access model documented.
  • Error-handling and incident path documented for failed automations.
